# TODO: Excel 问题分类工作流

本文档记录当前已确认的业务需求、项目结构调整方向和下一步实现计划。后续重新开启对话时，应先阅读本文档，再继续编码。

## 1. 项目目标

本项目用于处理 `input/` 目录下目标 Excel 文件中指定 sheet 的问题清单。

核心目标：

1. 根据配置找到目标 Excel 文件和需要处理的 sheet。
2. 读取每个目标 sheet 中第 3 行表头、第 4 行开始的数据。
3. 针对 E 列 `问题描述及原因初步判定` 的每行内容进行分类。
4. 分类结果后续写入输出副本的新列，不覆盖原始文件。
5. 当前阶段先生成“分类口径草案”，由用户确认后再执行逐行分类和写回 Excel。

## 2. 已确认需求

### 2.1 Excel 文件选择

- `input/` 下的 Excel 文件名会随日期更新，例如：
  - `2026-04-28_质保问题清单-汇总_192335.xlsx`
  - `2026-05-01_质保问题清单-汇总_165259.xlsx`
- 需要支持通配符或关键词匹配。
- 如果匹配到多个文件，按文件名日期选择最新文件。
- 如果文件名日期相同，再用修改时间作为兜底排序。

当前计划配置示例：

```dotenv
EXCEL_WORKBOOK_SHEETS_JSON={"*质保问题清单-汇总*.xlsx":["01 土建","02 机电-水","03 机电-暖"]}
```

注意：

- 目前 `common.env` 中已添加 `EXCEL_WORKBOOK_SHEETS_JSON`。
- `config.yaml` 中 `flows.excel_ai.workbooks` 从 `${EXCEL_WORKBOOK_SHEETS_JSON:-}` 读取。
- `config_loader.py` 已支持将 JSON 字符串解析为 dict/list。
- 之前发现 `${VAR:-{}}` 会和现有 env 插值正则冲突，不要在 YAML 默认值中直接写 `{}`。

### 2.2 目标 sheet

- 只处理 `EXCEL_WORKBOOK_SHEETS_JSON` 字典中指定的 sheet。
- sheet 名称未来会保持稳定。
- 不同 sheet 可以使用不同的分类标签体系。

### 2.3 表格结构

- 每个明细 sheet 的表头固定在第 3 行。
- 数据从第 4 行开始。
- E 列字段名为：`问题描述及原因初步判定`。
- 后续写回时，不固定 S 列。
- 后续写回时，应自动找到表格最后一列，并新增一列。
- 新列表头名称：`问题分类`。
- 输出文件保存到 `output/`，命名类似：

```text
原文件名_分类结果.xlsx
```

### 2.4 行处理规则

- 从第 4 行开始处理。
- 如果整行为空：跳过。
- 如果 E 列有内容：正常参与分类。
- 如果行有内容但 E 列为空：也需要处理，并在分类结果中标注，例如 `缺少问题描述`。
- 每行分类只允许一个类别。
- 后续真正逐行分类时，每行单独调用本地 AI。
- 单行分类时，除了 E 列，也要带同一行其他列作为上下文。

### 2.5 分类口径

当前阶段先不写回 Excel，也不输出逐行分类结果。

先生成“分类口径草案”：

- 按 sheet 汇总 E 列内容和同一行上下文。
- 调用本地 AI 为每个 sheet 生成候选中文分类标签。
- 草案保存为 Markdown 文件，例如：

```text
output/category_skills_draft.md
```

用户确认后，再把分类口径固化成配置或 skill 文档，然后进入逐行分类和写回阶段。

用户已提示的候选分类方向：

- 污水提升泵问题
- 隔油池装置问题
- 空调机组问题
- 消防风机问题
- 排风机问题
- 风阀问题
- 空调水阀问题
- 能量表问题

分类口径设计原则：

- 分类标签输出中文。
- 优先识别“设备/系统对象”作为分类标签。
- 不同 sheet 可以有不同标签体系。
- 若无法识别设备对象，再使用通用兜底分类，例如：
  - 管线问题
  - 阀门问题
  - 电气控制问题
  - 安装质量问题
  - 资料/手续问题
  - 缺少问题描述
  - 其他问题

## 4. 建议项目结构调整

当前三层结构保留：

```text

```

建议新增/调整如下。

### 4.1 新增模块：excel_target_resolver.py

路径：

```text
src/localai/modules/excel_target_resolver.py
```

职责：

- 解析 `flows.excel_ai.workbooks` 字典。
- 支持文件名精确匹配和通配符匹配。
- 在 `input/` 中查找匹配的 workbook。
- 按文件名日期选择最新文件。
- 日期格式优先识别文件名前缀 `YYYY-MM-DD`。
- 日期相同时按文件修改时间兜底。
- 输出目标对象列表：

```python
ExcelTarget(
    pattern=str,
    workbook_path=Path,
    sheet_name=str,
)
```

### 4.2 新增模块：excel_row_extractor.py

路径：

```text
src/localai/modules/excel_row_extractor.py
```

职责：

- 基于 workbook path 和 sheet name 读取结构化行。
- 表头行固定第 3 行。
- 数据从第 4 行开始。
- 读取整行上下文。
- 识别 E 列内容。
- 判断整行是否为空。
- E 列为空但整行有内容时，标记 `missing_problem_description=True`。
- 输出行对象：

```python
ExcelIssueRow(
    workbook_path=Path,
    sheet_name=str,
    row_number=int,
    problem_text=str,
    row_context=dict[str, str],
    missing_problem_description=bool,
)
```

建议：

- 仍使用 `openpyxl`。
- 对大文件使用 `read_only=True`。
- 表头字段为空时，可以用列字母或 `未命名列_X` 作为 key。

### 4.3 新增模块：category_prompt_builder.py

路径：

```text
src/localai/modules/category_prompt_builder.py
```

职责：

- 构造“分类口径草案” prompt。
- 按 sheet 分组输入样本行。
- 注入用户提供的种子分类标签。
- 要求 AI 输出中文分类标签。
- 要求每个标签包含：
  - 分类标签
  - 定义
  - 适用关键词
  - 示例行号
  - 注意事项

### 4.4 新增模块：category_skill_writer.py

路径：


## 6. 配置建议

建议在 `config.yaml` 中补充：

```yaml
flows:
  excel_ai:
    workbooks: ${EXCEL_WORKBOOK_SHEETS_JSON:-}
    header_row: ${EXCEL_HEADER_ROW:-3}
    data_start_row: ${EXCEL_DATA_START_ROW:-4}
    problem_column: ${EXCEL_PROBLEM_COLUMN:-E}
    category_header: ${EXCEL_CATEGORY_HEADER:-问题分类}
    draft_output_path: ${CATEGORY_DRAFT_OUTPUT_PATH:-./output/category_skills_draft.md}
    seed_categories: ${CATEGORY_SEED_CATEGORIES_JSON:-[]}
```

建议在 `common.env` 中补充：

```dotenv
EXCEL_WORKBOOK_SHEETS_JSON={"*质保问题清单-汇总*.xlsx":["汇总记录"]}
EXCEL_HEADER_ROW=3
```
